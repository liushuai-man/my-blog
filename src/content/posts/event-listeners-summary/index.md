感觉事件监听类型很多,所以去搜了一下做笔记,用来复习。

在 JavaScript 中，事件监听的类型非常丰富，覆盖了用户交互、文档状态变化、浏览器行为等多种场景。不同类型的事件对应不同的用户操作或系统触发时机，核心可按 交互场景 分为以下几大类，每类包含常用的事件类型及使用场景：

#### 一、鼠标相关事件（最常用）

针对鼠标的点击、移动、按下 / 松开等操作，是页面交互中最基础的事件类型。

| 事件类型    | 触发时机                               | 常用场景                                       |
| ----------- | -------------------------------------- | ---------------------------------------------- |
| click       | 鼠标左键单击元素（按下并松开）         | 按钮点击、导航跳转、指示器切换（如你的轮播图） |
| dblclick    | 鼠标左键双击元素                       | 双击放大图片、双击编辑文本                     |
| mousedown   | 鼠标任意按键按下（不区分左右键）       | 拖拽功能的开始（如拖拽元素）                   |
| mouseup     | 鼠标任意按键松开                       | 拖拽功能的结束                                 |
| mousemove   | 鼠标在元素上移动时（持续触发）         | 悬浮提示、拖拽跟随、鼠标轨迹绘制               |
| mouseenter  | 鼠标从元素外部进入元素内部时（不冒泡） | 导航菜单 hover 显示下拉列表、轮播图显示箭头    |
| mouseleave  | 鼠标从元素内部离开到外部时（不冒泡）   | 导航菜单 hover 隐藏下拉列表、轮播图隐藏箭头    |
| mouseover   | 鼠标进入元素或其后代元素时（冒泡）     | 类似 mouseenter，但子元素会触发父元素事件      |
| mouseout    | 鼠标离开元素或其后代元素时（冒泡）     | 类似 mouseleave，但子元素会触发父元素事件      |
| contextmenu | 鼠标右键单击（触发上下文菜单前）       | 自定义右键菜单（需阻止默认行为）               |

#### 二、键盘相关事件

| 事件类型 | 触发时机                                       | 常用场景                              |
| -------- | ---------------------------------------------- | ------------------------------------- |
| keydown  | 键盘任意按键按下（持续触发，包括功能键）       | 监听快捷键（如 Ctrl+C）、游戏控制     |
| keyup    | 键盘任意按键松开                               | 验证输入内容（松开按键后检查）        |
| keypress | 按下可打印字符的键（如字母、数字，已逐步废弃） | 旧版输入实时监听（建议用 input 替代） |

注意：keypress 无法识别功能键（如 Shift、Ctrl），且在现代浏览器中逐渐被 keydown + 字符判断替代。

#### 三、表单相关事件

针对表单元素（如输入框、按钮、下拉框）的交互，是表单验证、提交的核心。

| 事件类型 | 触发时机                                   | 常用场景                                   |
| -------- | ------------------------------------------ | ------------------------------------------ |
| input    | 输入框内容变化时（实时触发，支持复制粘贴） | 实时搜索提示、输入长度限制                 |
| change   | 元素值变化且失去焦点时（非实时）           | 下拉框（select）选择后触发、复选框状态切换 |
| focus    | 元素获得焦点时（如点击输入框、Tab 切换）   | 输入框获得焦点时高亮边框、显示提示         |
| blur     | 元素失去焦点时（如点击其他区域）           | 表单验证（失去焦点后检查输入合法性）       |
| submit   | 表单提交时（点击提交按钮或按 Enter）       | 表单整体验证（阻止非法提交）               |
| reset    | 表单重置时（点击重置按钮）                 | 重置后恢复默认状态提示                     |

#### 四、文档 / 窗口相关事件

针对文档加载、窗口大小变化、滚动等浏览器级别的行为，常用于页面初始化、响应式适配。

| 事件类型         | 触发时机                                   | 常用场景                               |
| ---------------- | ------------------------------------------ | -------------------------------------- |
| DOMContentLoaded | 文档 DOM 结构加载完成（无需等待图片、CSS） | 页面初始化代码（如绑定事件、渲染数据） |
| load             | 整个页面（包括图片、CSS、脚本）加载完成    | 图片懒加载初始化、页面加载进度完成     |
| beforeunload     | 页面即将关闭 / 刷新时（如关闭标签页）      | 提示用户 "是否放弃未保存内容"          |
| resize           | 浏览器窗口大小改变时（持续触发）           | 响应式布局调整（如重新计算元素尺寸）   |
| scroll           | 页面或元素滚动时（持续触发）               | 滚动加载更多数据、导航栏吸顶效果       |
| unload           | 页面完全卸载后（如关闭标签页）             | 清理资源（如断开 WebSocket 连接）      |

#### 五、触摸相关事件（移动端）

针对移动设备的触摸操作（如手指点击、滑动），适配移动端交互场景。

| 事件类型    | 触发时机                         | 常用场景                           |
| ----------- | -------------------------------- | ---------------------------------- |
| touchstart  | 手指触摸屏幕时                   | 移动端点击、拖拽开始               |
| touchmove   | 手指在屏幕上滑动时（持续触发）   | 移动端轮播图滑动、下拉刷新         |
| touchend    | 手指离开屏幕时                   | 滑动结束后切换轮播图、计算滑动距离 |
| touchcancel | 触摸被中断时（如来电、弹出通知） | 中断后恢复初始状态                 |

注意：触摸事件会与鼠标事件存在 “延迟触发”（约 300ms，为了判断是否是双击），可通过 meta name="viewport" content="user-scalable=no" 取消延迟。

#### 七、事件监听的通用使用方式

无论哪种事件类型，都通过 addEventListener 绑定，语法统一：

```
// 语法：元素.addEventListener(事件类型, 事件处理函数, 可选参数)
const btn = document.querySelector('button');

// 1. 鼠标点击事件
btn.addEventListener('click', function() {
  console.log('按钮被点击了');
});

// 2. 输入框内容变化事件
const input = document.querySelector('input');
input.addEventListener('input', function() {
  console.log('输入内容：', this.value);
});

// 3. 页面加载完成事件
document.addEventListener('DOMContentLoaded', function() {
  console.log('DOM结构加载完成，可以绑定事件了');
});
```

#### **总结**

事件类型的选择需根据 **交互场景** 决定：

- 鼠标操作 → 选鼠标事件（如 click、mouseenter）；
- 表单输入 → 选表单事件（如 input、blur）；
- 页面初始化 → 选文档事件（如 DOMContentLoaded）；
- 移动端交互 → 选触摸事件（如 touchstart、touchmove）。

掌握常用事件类型的触发时机，是实现流畅页面交互的基础。
